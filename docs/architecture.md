There are five modules inside the app. `capture` grabs frames and feeds them to `pipeline`, which runs YOLOv8 detection, MobileNetV2 feature extraction, and TransOBCD/ConvOBCD classification. When the pipeline produces a DetectionResult, `alarm` routes it: local alerts go to `ui`, remote alerts go to the background service. Every other module reads `config` at startup for YAML settings.
